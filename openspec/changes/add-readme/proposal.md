## Why

O projeto não possui documentação de entrada. Um README.md é essencial para que outros desenvolvedores (e eu mesmo no futuro) entendam o que é o sistema, como foi construído e como contribuir — especialmente considerando que a abordagem de desenvolvimento é atípica: Spec-Driven Development com ferramentas de IA.

## What Changes

- Criação do arquivo `README.md` na raiz do repositório
- Documentação do projeto: propósito, stack, arquitetura
- Explicação didática de Spec-Driven Development (SDD) e como o OpenSpec o implementa
- Diagrama visual dos módulos e domínio do sistema
- Guia de contribuição seguindo o fluxo SDD
- Menção às ferramentas de IA utilizadas (OpenCode, Big Pickle/Codex, RTK)

## Capabilities

### New Capabilities

- `project-documentation`: Documentação pública do projeto em README.md, incluindo visão geral, arquitetura, fluxo de desenvolvimento e guia de contribuição

### Modified Capabilities

(nenhuma — esta change é puramente documental)
